Предварительная настройка Git

Настройки Git производятся командой

$ git config -[параметр]

Все параметры команды config можно просмотреть здесь


Установка имени пользователя

При превоначальной установке Git в первую очередь указывается идентификатор пользователя и адрес электронной почты. Это необходимо по причине того, что Git вносит эту унформацию в кажый сделанный вами коммит.

Установка имени пользователя в Git производится командой

$ git config --global user.name "[Ваше имя]"

В моем случае комманда будет выглядеть так:

$ git config --global user.name "Kirill Silianov"

Установка адреса электронной почты

Установка адреса электронной почты в Git производится командой

$ git config --global user.email [e-mail адрес]

В моем случае комманда будет выглядеть так:

$ git config --global user.email admin@siada.ru

Установка индивидуальных учетных данных для репозитория

После ввода идентификационной информации Git будет подписывать ею все действия пользователя. Если для конкретного репозитория необходимо использовать другие учетные данные, то необходимо передти в директорию проекта и выполнить команду без параметра "--global". Пример:

$ cd c:/projects/
$ git config user.name "Vasya Pupkin"
$ git config user.email info@siada.ru

Выбор текстового редактора для Git

Тектовый редакторв в Git используется в случае, если вам необходимо набрать какое либо сообщение. Если не определить этот параметр, Git будет использовать текстовый редактор по-умолчанию, установленный в системе. Для Windows это "Notepad", а для Linux "Vim" или "Emacs". Редактор по умолчанию устанавливается следующей командой:

$ git config --global core.editor [имя редактора или "путь к исполняемому файлу редактора"]

Например:

$ git config --global core.editor vim
$ git config --global core.editor "C:\Program Files (x86)\Notepad++"

Выбор программы для сравнения текстовых файлов

Программа для сравнения файлов используется в тех случаях, когда требуется определить различия между версиями файлов или когда возникает конфликт слияния текстовых файлов. Для установки программы сравнения файлов исползуется параметр конфигурации "megre.tool [имя программы или "путь к исполняемому файлу программы"]". Пример комманды:

$ git config --global megre.tool kdiff3
$ git config --global core.editor "C:\Program Files (x86)\Diffuse\diffusew.exe"

Просмотр настроек Git

Для просмотра настроек Git используется команда конфигурации "--list". Например следующая команда:

$ git config --list

Выведет весь список настроек

core.symlinks=false
core.autocrlf=true
core.fscache=true
color.diff=auto
color.status=auto
color.branch=auto
color.interactive=true
help.format=html
http.sslcainfo=C:/Program Files/Git/mingw64/ssl/certs/ca-bundle.crt
diff.astextplain.textconv=astextplain
rebase.autosquash=true
credential.helper=manager
filter.lfs.clean=git-lfs clean %f
filter.lfs.smudge=git-lfs smudge %f
filter.lfs.required=true
user.name=Kirill Silianov
user.email=admin@siada.org
core.editor=C:\Program Files (x86)\Notepad++
megre.tool=C:\Program Files (x86)\Diffuse\diffusew.exe

Для получения значения определенного параметра необходимо указать параметр команды "config"

$ git config --list user.name

Вернет:

Kirill Silianov

Задания

  1. Просмотрите значения настроек до их изменения
  2. Установите имя пользователя в Git
  3. Установите адрес электронной почты в Git
  4. Укажите текстовый редактор, используемый по умолчанию в Git
  5. Ускажите инструмент для сравнения и слияния текстовых файлов
  6. Просмотрите значения настроек после изменения
