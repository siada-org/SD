Общие сведения

Использовать Git можно различными способами. Для Git было создано множество графических интерфейсов, фронтэнодов и прочих инструментов, упрощающих работу с Git. Эти инструменты будут рассмотрены в сответсвующих дополнительных разделах, а в ходе данного курса, для управления и настройки Git будут использоваться консольные команды. Причин этому много:

  • консоль - единственный способ выполнения всех команд Git;
  • графические интерфейсы пользователя (GUI), в угоду простоте использования, имеют урезанный функционал;
  • GUI в сравнении с командной строкой гораздо ниже по производительности, что немаловажно при работе с большими проектами;
  • команды консоли унифицированы для всех версий Git и независимы от операционной системы, GUI же отличаются друг от друга как по функциональности, так и по способу управлению ими;
  • знание консольных команд позволяет понять механику работы Git и позволит без проблем использовать в последствии любой GUI, посколько все GUI являются надстройками над консолью, интерпретирующими действия пользователя в команды консоли;

2.2. Установка Git - выбор пути установки

Первые два этапа стандартые для установки программ

На третьем этапе необходимо выбрать компоненты для установки:

  • additional icons - дополнительные иконки приложения и ярлыки рабочего стола;
  • Windows Explorer integration - позволяет добавить два пункта к контекстному проводника:
    • Git Bash Here - добавляет к контекстному меню пункт "Git Bash here" при выборе которого открывается консоль с выбранной директорией
    • Git GUI Here - добавляет к контекстному меню пункт "Git GUI here" при выборе которого запускается GUI Git с выбранной директорией
  • Associate .git* configuration files with the default text - установить, используемый по умолчанию в системе текстовый редактор в качестве текстового редактора по умолчанию в Git;
  • Associate .sh files to be run with Bash - ассоциировать файлы с расширением *.sh с консолью "Bash";
  • Use a TrueType font in all console windows - использовать шрифты TrueType в консоли CMD;

2.3. Установка Git - выбор компонентов

Следующим этапом следует выбор папки в меню "Пуск" в которой будут располагаться ярлыки Git.

Следующим пунктом установки будет выбор способа использования командной сторки Git. У нас есть выбор из трех пунктов:

  • Use Git from Git Bash only - использовать только эмулятор Bash для управления Git;
  • Use Git from the Windows Command Promt - использовать для работы с Git как Bash так и CMD;
  • Use Git and optional Unix tools from the Windows Command Promt - добавить возможность использования в CMD некоторых инструментов консоли Unix (не рекомендуется);

2.4. Установка Git - выбор способа использования командной строки в Git

Затем следует пункт обработки концов строк текста описания при commit или checkout. Для выбора доступно три пункта:

  • Checkout Windows-style, commit Unix-style line ending - для checkout используются окончания сторк как в Windows, а для коммитов используются окончания строк Unix;
  • Checkouts as-is, commit Unix-style line endings - окончания строк checkout спользуются "Как есть", а строки commit переделываются в Unix-стиле;
  • Checkouts as-is, commit as-is - checkout строки используются "как есть", commit строки используются "как есть";

2.5. Установка Git - выбор способа окончания строк

Следующий пункт - использования эмулятора bash. Данный этап имеет два пункта доступных для выбора:

  • Use MinTTY (The default terminal of MSYS2) - Использовать в качестве эмулятора Bash MinTTY;
  • Use Windows default console window - Использовать в качестве эмулятора Bash консоль Windows;

Установка GitHub для Windows

Другим вариантом установки Git в операционной системе Windows является установка GitHub desktop для Windows. Данный пакет включает в себя как версию с коммандной строкой так и графический интерфейс пользователя. Помимо этого GitHub desktop включает в себя оснастку для работы с Git посредством консоли Powershell

Загрузка дистрибутива доступна по адресу desktop.github.com. При заргрузке скачивается файл маленького размера, который после запуска скачивает все необходимые данные с Интернета
